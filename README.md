# MetricsOracle

MetricsOracle — инструмент для статистического анализа временных рядов метрик. Загрузите `.txt` или `.xlsx` файл, и система автоматически определит аномалии через Z-Score или IQR, назначит светофорный статус каждой метрике и построит линейный прогноз на 24 периода. Встроенный демо-режим позволяет оценить возможности без загрузки собственных данных.

## Команды

```bash
npm run dev       # запустить dev-сервер (http://localhost:3000)
npm run build     # production-сборка
npm run test      # тесты в watch-режиме
npm run test:run  # тесты однократно
```

## Формат входных данных

Файл `.txt` — TSV (табуляция как разделитель) или CSV (запятая):

```
period      bug_ratio   wip_count   cycle_time
2024-W01    0.12        8           14.5
2024-W02    0.15        10          16.2
2024-W03    0.18        12          17.8
```

- Первая строка — заголовки: первая колонка — метка периода, остальные — названия метрик
- Первая колонка каждой строки — период (произвольная строка: `2024-W01`, `Jan`, `Q1` и т.п.)
- Все остальные колонки — числовые значения метрик
- Минимум 5 строк данных (рекомендуется 8+)
- Файл `.xlsx`: та же структура, первый лист

## Демо-режим

Нажмите «Запустить демо» для анализа 24 недель синтетических данных. Включает 8 метрик:

| Метрика | Описание |
|---|---|
| `prod_bug_ratio` | Доля ошибок в продакшне |
| `wip_testing` | Задачи в тестировании (WIP) |
| `bug_lifetime_days` | Среднее время жизни баги |
| `release_delivery_speed` | Скорость поставки релиза |
| `test_debt_ratio` | Накопленный тестовый долг |
| `debt_burndown` | Скорость сжигания долга |
| `testing_cycle_time` | Время цикла тестирования |
| `automation_coverage` | Покрытие автотестами |

Данные содержат управляемый пик нагрузки на неделях 14–15 и линейный рост `test_debt_ratio`, что позволяет наблюдать жёлтые и красные зоны.

## Алгоритмы

- **Z-Score** — при нормальном распределении (|skewness| ≤ 1). Зоны: green `< 1σ`, yellow `1–3σ`, red `> 3σ`
- **IQR** — при скошенном распределении. Зоны: green внутри `[Q1, Q3]`, yellow выход до 2 единиц, red выход более 2 единиц
- **Светофор**: red при ≥ 3 метриках в красной зоне; yellow при ≥ 3 в жёлтой/красной; иначе green
- **Прогноз**: линейная регрессия (МНК), экстраполяция на 24 периода для определения смены зоны
